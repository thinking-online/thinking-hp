# 家勉部リーグ機能 設計書 v1

家勉部に「予選リーグ(毎週シャッフル)」と「全国大会(全体トップ5)」を追加する。
この文書が仕様の正典。実装(API・ポータル)は全てこれに従う。

---

## 1. 全体像

2つの競争を同時に見せる。

- **予選リーグ**: 全員を5人前後のブロックにランダム配置。ブロック内で週間勉強時間を競う。毎週月曜0時台に自動で再シャッフル。段位・昇格降格はなし(将来拡張)。
- **全国大会**: ブロック横断で、全員の週間勉強時間トップ5を公開。全体の頂点を見せる枠。

近い目標(自分のブロックで上位)と高い目標(全国トップ5)が両立する二段構え。

---

## 2. 予選リーグの仕様

### ブロック分け
- 目標人数は1ブロック5人。
- 対象は毎週シャッフル時点で「有効」「仮入部」の会員のみ。
- 余りは均等配分(ルールA): 総人数からブロック数を決め、余りを各ブロックに1人ずつ散らす。3人ブロックのような寂しい塊を作らない。
  - 計算: ブロック数 = round(人数 / 5)。最低1。人数をブロック数で割り、余りを先頭のブロックから1人ずつ配る。
  - 例: 40人 → 8ブロック×5人。37人 → round(7.4)=7ブロック → 各5〜6人(5,5,5,5,5,6,6 等)。42人 → 8ブロック → 各5〜6人。
- ブロック番号は 1,2,3… の整数。会員マスタに保存。

### シャッフルのタイミング
- 毎週月曜の0時台(GASの週次トリガー: 毎週月曜・午前0〜1時)。
- GASのトリガーは分単位の正確な指定ができないため、実際は月曜0:00〜1:00の間に実行される。日曜23:59で週が締まった直後に走る、という理解で運用上問題ない。
- 処理内容: 前週を確定 → 対象会員を集める → シャッフル → 新ブロック番号を一斉に書き込み → 週情報を更新。

### 週の途中の出入り
- 途中入会: その週は未配置。翌週のシャッフルから参加。
- 途中退会: ブロック表示にはその週いっぱい残る(週末まで固定)。翌週から外れる。

### 順位
- ブロック内を週間勉強時間の降順で並べる。
- 同じ時間は同順位(同率)。次の順位は人数分飛ばす(1,1,3 形式)。
- 記録ゼロの人も表示する(0分で最下位)。

---

## 3. 全国大会の仕様

- ブロック関係なく、全会員の週間勉強時間トップ5を公開。
- 表示は順位・表示名・勉強時間。トップ5の中に自分がいればハイライト。
- **自分がトップ5外のとき**: 「あなたの上にあと◯人」という表示にする(全体母数は見せない)。
  - 母数が小さいうちは人数を隠すことで「ショボさ」を避ける狙い。
  - 将来、母数が増えたら「全国◯位」表示へ切り替えられるよう、設定で出し分け可能にする(設定キー: 全国順位表示モード = hidden / rank)。初期は hidden。

---

## 4. データ設計(会員マスタへの追加列)

01_会員マスタに以下を追加(末尾列に追加。既存列の順序は変えない):

- **ブロック番号**: 今週のブロック番号(整数)。未配置は空。

週情報は06_設定で管理:
- **リーグ週**: 現在シャッフル中の週ID(例 2026-W25)。週次バッチが更新。
- **ブロック目標人数**: 初期値 5。後から調整可能。
- **全国公開人数**: 初期値 5(トップ5)。
- **全国順位表示モード**: hidden(初期) / rank。

## 5. API(v5)エンドポイント

### 既存に追加
- `league`(GET, 要token): 自分のブロックのメンバー一覧と各自の週間勉強時間、自分のブロック内順位を返す。
- `nationals`(GET, 要token): 全国トップ5(順位・表示名・分)を返す。自分がトップ5外なら above_count(自分の上に何人いるか)を返す。表示モードが rank なら my_rank も返す。

### league レスポンス例
```
{
  "status": "ok",
  "week": "2026-W25",
  "block_no": 3,
  "members": [
    { "rank": 1, "name": "ゆうき", "minutes": 920, "is_me": false },
    { "rank": 2, "name": "てった", "minutes": 540, "is_me": true },
    ...
  ],
  "my_rank": 2,
  "block_size": 5
}
```

### nationals レスポンス例(トップ5外の場合)
```
{
  "status": "ok",
  "week": "2026-W25",
  "top": [
    { "rank": 1, "name": "ゆうき", "minutes": 920 },
    ... 5人分
  ],
  "in_top": false,
  "above_count": 7,        // 自分の上に7人(=実質8位だが母数は出さない)
  "my_minutes": 380,
  "mode": "hidden"
}
```

## 6. 週次バッチ(GAS)

関数 `weeklyLeagueShuffle()`:
1. 現在の週IDを計算(日曜始まり週。月曜0時台実行なので「今日が属する週」)。
2. 01_会員マスタから「有効」「仮入部」の line_user_id を集める。
3. 配列をシャッフル(Fisher–Yates)。
4. ブロック目標人数からブロック数を算出し、均等配分でブロック番号を割り当て。
5. 会員マスタのブロック番号列を一括更新(対象外の人は空に)。
6. 06_設定「リーグ週」を新週IDに更新。

トリガー: 編集 → 現在のプロジェクトのトリガー → トリガーを追加 → 関数 weeklyLeagueShuffle / 時間主導型 / 週ベースのタイマー / 毎週月曜 / 午前0時〜1時。
※ てった側の手作業はこのトリガー登録(1回・5分)のみ。手順は別途。

## 7. ポータル表示(Cursorで実装)

配置: 既存の「勉強時間ランキング」セクションを、リーグ表示に発展させる。

- **予選リーグ**: 「予選リーグ」見出し。自分のブロック5人前後をランキング表示(順位・表示名・時間バー)。自分の行をゴールドでハイライト。大会っぽい雰囲気。
- **全国大会**: 「全国大会」見出し。トップ5を表彰台風に(1位を大きく)。自分がいればハイライト、いなければ「あなたの上にあと◯人 — ここまであと◯分」と煽る。
- 既存の全体Top10ランキングはリーグに置き換える(役割が重複するため)。小テストランキングはそのまま残す。
- データが来ない/旧APIのときは従来表示にフォールバックし、エラーで画面を壊さない。

デザイン: 紺#172642 / ゴールド#C8A55A / 紙#F6F4EF。大会感を出すが既存トーンを壊さない。
