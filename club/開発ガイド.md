# 家勉部ポータル 開発ガイド

このファイルは club/ 配下を変更するAI(Cursor等)と開発者のための設計書。
作業前に必ず全文を読むこと。

## 0. 大原則

仕様の正典は「実際に動いているポータルとAPIとDB」である。
LP (https://thinking-online.com/study-room) は目指す方向の参考であり、確定仕様ではない。
LPに書いてある機能がポータルに無くても、勝手に実装しないこと。
機能追加は必ず運営者(てった)の指示があったものだけを行う。
方針: 最低限から始めて、運用しながら一つずつ足す。

## 1. システム概要

家勉部は、オンライン塾THINKING(合同会社ARC)が運営する月額制オンライン部活。
このリポジトリにあるのは会員専用ポータル(部室)のフロントエンドのみ。

全体は3層構造:

1. データ層: Googleスプレッドシート「【家勉部】DB」(このリポジトリには無い)
2. ロジック層: Google Apps Script「家勉部システム」をWebアプリとしてデプロイしたAPI(このリポジトリには無い。Apps Scriptエディタで管理)
3. 表示層: club/index.html (このリポジトリで管理。Netlifyで thinking-online.com/club/ に自動デプロイ)

ユーザーはLINEのLIFF経由でポータルを開く。LIFF URL: https://liff.line.me/2010378695-4j2cx38C

## 2. 絶対に変更してはいけない箇所

club/index.html の script 内にある以下は、デザイン変更の依頼であっても変更禁止:

- 定数 API_URL と LIFF_ID の値
- liff.init / liff.login / liff.getIDToken / liff.getDecodedIDToken の呼び出しフロー
- api() 関数、submitLog() 内の fetch のURL組み立て・メソッド・bodyの形式
- APIに送るパラメータ名 (action, token, debug_user, minutes, date)
- APIレスポンスのJSONキー名への依存箇所(キー名を別名に変えない)

POSTはContent-Typeを指定しない(text/plainのまま)。指定するとCORSのpreflightが発生してGASで失敗する。

HTML構造とCSSは自由に変更してよい。ただし既存のid属性はJSが参照しているため、idを変更・削除する場合は参照しているJSも整合させること。

## 3. 認証とゲートの仕組み

1. ページを開くとLIFFが初期化され、LINEのIDトークンを取得
2. 全API呼び出しに token としてIDトークンを付与
3. GAS側がLINEの検証API(client_id=LIFF_CHANNEL_ID)でトークンを検証し、line_user_idを特定
4. 01_会員マスタと照合し、ステータスで出し分ける
   - 有効: 全機能
   - 仮入部: 限定コンテンツ以外(コンテンツは公開対象=仮入部も可のみ)
   - 休会/退会/未登録: ゲート画面(JSONは {status:"denied", reason:"..."})
5. 未登録者のゲート画面には本人のLINE ID(IDトークンのsub)とコピー導線を表示する。
   これが入部フロー(IDを公式LINEに送る→運営がマスタに登録)なので消さないこと

## 4. API仕様

ベースURL(API_URL): GASのWebアプリ /exec

GET (クエリ: action, token):

- action=ping → {ok, version, time}
- action=me → {status:"ok", member:{member_id, display_name, grade, member_status, title}, today_minutes, week_minutes, streak, total_days, rank, room_name, room_id_set}
  room_name: Zoom自習室の入室名「LINE表示名｜識別番号」。room_id_set: 識別番号(会員マスタO列)が設定済みかの真偽値
- action=weekly → {status:"ok", week(次回テストの日付ラベル 例"6/21(日)"), test_date(yyyy-mm-dd), items:[{subject, range, note}], room_url, hours}
  v11以降、小テスト範囲は別スプレの【縦型】小テストスケジュールから「今日以降で最も近い日曜」の分を自動取得(日曜当日はその日、月曜で次回へ切替)。スプレIDとシート名は06_設定で管理。旧08_週間情報は不使用
- action=ranking (勉強時間・今週・日曜始まり) → {status:"ok", week_start, ranking:[{rank, name, minutes, is_me}], my_rank, my_minutes, total}
- action=test_ranking (小テスト・掲示板スプレ直読み) → {status:"ok", period, subjects:[{subject, entries:[{rank, name, rate, is_me}]}]}
- action=contents → {status:"ok", items:[{title, url, category, added}]}

POST (body: JSON文字列):

- {action:"log_study", token, minutes, date?} → {status:"ok", date, minutes, overwritten, today_minutes, week_minutes}
  同一ユーザー・同一日付は上書き(二重計上防止)。dateは "yyyy-MM-dd"、省略時は今日

追加GET(v5以降):

- action=league (予選リーグ・自分のブロック) → {status:"ok", assigned, week, block_no, members:[{rank, name, minutes, is_me}], my_rank, block_size}
  assigned=false のときはまだ今週のブロック未配置(案内表示。ランキングは出さない)
- action=nationals (全国大会・全体トップN) → {status:"ok", week, top:[{rank, name, minutes, is_me}], in_top, my_minutes, mode, above_count?, my_rank?}
  in_top=false かつ mode="hidden" のとき above_count(自分の上の人数)。mode="rank" のとき my_rank。全体母数は出さない

追加POST(v6以降):

- {action:"coach", token, message, history:[{role, content}]} → {status:"ok"|"limit"|"error", reply}
  AIコーチ「家勉くん」。messageは今の発言、historyは直近の会話(最新が後ろ)。
  reply は status に関わらず家勉くんの台詞として表示してよい。
  APIキーはGASのスクリプトプロパティ(ANTHROPIC_API_KEY)に保管。HTML/JSには絶対に書かない。

宣言機能(v7以降):

- GET action=declarations (クエリに scope=league|all を追加) → {status:"ok", date, scope, total_today, i_declared, my_text, my_goal, declarations:[{name, text, goal_minutes, cheers, is_me, cheered_by_me, target}]}
  target はファイト送信専用の識別子。画面には表示しない(名前のみ表示)。
- POST {action:"declare", token, text, goal_minutes?} → {status:"ok", updated} (今日の宣言を投稿/上書き。1日1宣言)
- POST {action:"cheer", token, target} → {status:"ok", cheered, count} (今日の宣言にファイト送信/取消のトグル)

エラー系: {status:"unauthorized"} / {status:"denied", reason} / {status:"error", reason}

## 5. DBシート構成(参考。リポジトリ外)

01_会員マスタ: member_id, line_user_id(主キー), 氏名, 表示名, 学年, プラン, ステータス(申請中/仮入部/有効/休会/退会), 仮入部期限, 入会日, 退会日, 決済ID, 保護者line_user_id, 称号, 備考, 識別番号, ブロック番号(リーグ用・週次更新), LINE表示名(申請時に自動記録), 照合コード(申請時に自動生成・生徒が公式LINEに送る), Lステップ番号(承認時に運営が手で記入・恒久的な照合キー), 本人確認(申請時に生徒が入力・生年月日など)
  ※ 申請(handleApply)はヘッダー名で列位置を決めて書き込むため、列の順序を変えても壊れない。新列は末尾に追加してよい。
02_入退室ログ / 03_テスト結果 / 04_週次集計 / 05_配信キュー
06_設定: key-value (自習室URL, 開放時間, 現在week_id, LIFF_CHANNEL_ID, LIFF_ID, ランキングスプレID, ランキングシート名, ランク定義, 星間隔, リーグ週, ブロック目標人数, 全国公開人数, 全国順位表示モード, 家勉くん上限_仮入部, 家勉くん上限_正式, 家勉くんモデル など)
  ※ 家勉くんの上限はステータス別(仮入部=家勉くん上限_仮入部・有効=家勉くん上限_正式)。数字を変えるだけで即反映(再デプロイ不要)
07_取込_DrOke / 08_週間情報(week_id, 科目, 範囲, 備考) / 09_コンテンツ(タイトル, URL, カテゴリ, 公開対象, 追加日, 表示順)
10_勉強時間ログ: timestamp, line_user_id, 日付, 分, source (追記専用・同日上書き)
13_家勉くん利用: 日付, line_user_id, 回数 (1日の利用上限管理。自動生成)
16_家勉くん会話ログ: 日時, 会員番号, 表示名, LINE表示名, 生徒の発言, 家勉くんの返事 (家勉くんとの全会話を記録。どんな相談が来ているか把握用。自動生成。量が増えたら古い行を別シートに退避してよい)
14_宣言: 投稿日時, line_user_id, 表示名, 日付, 宣言, 目標分, 状態(表示/非表示), ファイト数 (今日の宣言。1日1宣言・上書き。状態を非表示にすると一覧から消える=運営の問題投稿対応)
15_ファイト: 日時, from_line_user_id, 宣言日付, 宣言主line_user_id (誰が誰の宣言にファイトしたか。トグル管理)

リーグの週次シャッフルは weeklyLeagueShuffle 関数を毎週月曜0時台にトリガー実行(有効/仮入部を5人前後のブロックに均等配分)。
家勉くんのAPIキーは ANTHROPIC_API_KEY としてスクリプトプロパティに保管(シートにもHTMLにも置かない)。

小テストランキングの実データは外部の「【公開用】THINKING掲示板」の【週間】週間ランキング👑シートをGASが直読みしている。

## 6. デザインシステム

ブランド: THINKING / 家勉部。トーン: ミニマルで知的、Apple/UNIQLO的。受験産業的な精神論・絵文字過多は避ける。

カラートークン(CSS変数):
- --navy: #172642 (基調)
- --navy-deep: #0F1A30
- --gold: #C8A55A (アクセント。多用しない)
- --gold-dark: #8F702E
- --paper: #F6F4EF (背景)
- --ink: #1C2433 / --muted: #6B7280 / --line: #E6E2D8

フォント: 本文 Noto Sans JP、数字 Space Grotesk(class="num")
レイアウト: モバイル前提、max-width 480px、カードUI

世界観の語彙(コピーを書くときはこの言葉を使う):
戦友 / 部員 / 部室 / ブースト時間 / 10時間チャレンジ / 一緒だから、続けられる。
キャッチコピーの参考はLP(0章の通り、内容は変わりうる)。

## 7. 更新フロー

見た目の変更: club/index.html を編集 → push → Netlifyが自動デプロイ(1〜2分)
機能の変更(API側): Apps Scriptエディタで編集 → デプロイを管理 → 新バージョンでデプロイ(URLは不変)。リポジトリ外の作業

## 8. 動作確認

- 本番: スマホのLINEで LIFF URL を開く
- ゲート確認: 01_会員マスタのステータスを変えてリロード(即時反映)
- ブラウザ単体テスト: 06_設定に「デバッグモード / ON」の行を足すと ?action=me&debug_user=xxx 形式でテスト可。終わったら行を削除かOFFに戻すこと(本番では必ず無効化)
