# 英文解釈 確認テスト制作プロジェクト

英文60個について、「訳の丸暗記」ではなく「構造・意味の本当の理解」ができているかを確認するテストを作るプロジェクト。

## チーム編成(制作フロー)

1問ごとに以下の役割を直列+並列で回す。

| 順 | 担当 | 仕事 | 成果物 |
|---|---|---|---|
| 0 | **設計担当** | 何を確認すれば「本当の理解」の証拠になるかを定義し、作成仕様書を作る(プロジェクト開始時に1回) | `design-spec.md` |
| 1 | **作成担当** | 対象文の「解釈上の山場」を特定し、仕様書どおりに1文3問を作る | 問題ドラフト |
| 2a | **一意性チェック担当**(並列) | 答えが1つに定まるか。別解・複数正解・採点で割れるグレー答案・設問文の曖昧さを意地悪に探す | 指摘リスト |
| 2b | **妥当性チェック担当**(並列) | 丸暗記で解けないか。「訳だけテスト」(英文を隠して配布訳だけで解けるか)のシミュレーション、消去法・メタ攻略耐性、山場のカバー、対象学年適合、誤答分析の質 | 指摘リスト |
| 3 | **編集担当(統合)** | 2a・2bの指摘を反映して最終版を作る。差し替え・大修正が入った問は再度チェックに回す | 最終版 |

## 全設問共通の合格基準

> **「英文を隠して配布訳だけを渡された生徒が正解できるか?」— Yes ならその問題は不合格。**

## 進め方

- いきなり量産せず、まず1問(→ `sample-01.md`)で型を固める。
- 型が固まったら5文ずつ試作 → チェック通過 → 残りに展開。
- 1文3問・3〜5分。60文なら10文×6回(1回約40分)に分割して運用。
