# スプレッドシート形式仕様(Googleフォーム量産用・横持ち形式)

全問客観式(4択)。**1行=1英文**。`questions-60.csv` をGoogleスプレッドシートにインポートし、`gas/form-generator.gs` の `createFormsFromSheet()` を実行すると、クイズ形式のGoogleフォーム(自動採点・解説フィードバック紐づけ)が一括生成される。

## シート名: `questions`

| 列 | 内容 |
|---|---|
| `問題No` | 英文の通し番号(1〜60) |
| `問題` | 対象英文(プレーンテキスト) |
| `訳` | 配布訳 |
| `問題01` | 設問文+選択肢(セル内改行区切り。設問文の後に `A. …` `B. …` `C. …` `D. …` の4行) |
| `問題01の答え` | 正解記号(A/B/C/D の1文字) |
| `問題01の解説` | 解説。フォームの解答フィードバック(正解時・不正解時共通)に表示。生徒向けの言葉で2〜3文 |
| `問題02`〜`問題04の解説` | 同様の3列セット。設問が少ない英文は空欄(GASが自動スキップ) |

- 1行目はヘッダー行(上記の列名と完全一致させること。GASは列名で参照する)。
- 「問題0X」セルの選択肢行は必ず `A. `(記号+ピリオド+スペース)で始める。GASはこの形式でパースする。
- CSVはUTF-8(BOM付き)。セル内改行はダブルクォートで保護されているので、Googleスプレッドシートの「ファイル > インポート」でそのまま取り込める。

## 品質基準(全設問共通)

1. 正解が1つに定まること(誰が採点しても同じ)。全問4択の客観式で、記述問題は使わない。
2. 「英文を隠して配布訳だけで解けるか?」→ Yes なら不合格。
3. 誤答肢は「ありがちな誤読」を反映(でたらめ禁止)。選択肢の長さ・体裁を揃え、メタ攻略(長い肢=正解、位置のカン等)を防ぐ。
4. 設問間で答えのリークがないこと。正解記号は問題間でばらす。
5. 問題型は言い換え選択(英→英)/用法識別(同じ使い方のif・that等)/構造識別/語法/動作主特定/論理関係を、その文の「解釈上の山場」に合わせて選ぶ。

## 運用フロー

1. 各英文は「作成担当 → 検証担当(一意性+訳だけテスト+メタ攻略耐性)」のチームを通過した確定版のみCSVに載せる。
2. CSVをスプレッドシートにインポート(シート名 `questions`)。
3. `gas/form-generator.gs` を Apps Script に貼り付けて `createFormsFromSheet()` を実行。
4. 10文=1フォームで分割生成され(60文なら6フォーム)、`forms` シートに編集用・回答用URLが書き戻される。
