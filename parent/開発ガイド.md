# THINKING保護者通信 — 開発・運用ガイド

保護者向け月刊通信サイトの追加・更新手順です。王様通信（`lp/king-letter-*`）とは別に、リポジトリ直下の `parent/` で管理します。

## 公開URL

| ページ | URL |
|--------|-----|
| アーカイブ（バックナンバー一覧） | `https://thinking-online.com/parent` |
| 各号 | `https://thinking-online.com/parent/YYYYMM-NN` |
| 例：6月号 | `https://thinking-online.com/parent/202506-01` |

旧URL `/parent-202506-01` は `/parent/202506-01` へ自動リダイレクトされます。

## フォルダ構成

```
parent/                        # 保護者通信（lp/ とは別フォルダ）
├── index.html                 # アーカイブ一覧
├── styles.css

│   ├── index.html
│   ├── styles.css

│   └── assets/img_cover.png
├── 202507-01/                 # 7月号（準備済み・未公開の例）
└── …
pdfs/
└── parent-YYYYMM-*.pdf        # まとめ・長文解説・コラムのPDF
```

## 公開・非公開の切り替え

**準備ができた号だけ**公開します。未公開の号はフォルダに置いておいても、次の2点をしない限りユーザーからは見えません。

1. `netlify.toml` にリダイレクトを追加しない
2. `parent/index.html` のアーカイブにカードを載せない

### 新しい号を公開するとき

1. 号の中身（本文・PDF）を仕上げる
2. `netlify.toml` にリダイレクトを追加（下記テンプレート）
3. `parent/index.html` の `issue-list` **一番上**にカードを追加
4. push → デプロイ

```toml
# THINKING保護者通信 7月号 — /parent/202507-01 で公開
[[redirects]]
  from = "/parent/202507-01"
  to = "/parent/202507-01/index.html"
  status = 200
[[redirects]]
  from = "/parent/202507-01/*"
  to = "/parent/202507-01/:splat"
  status = 200
```

**現在公開中：** 6月号のみ（`/parent`・`/parent/202506-01`）

## 毎月の追加手順（約30分）

### 1. 新しい号フォルダを作る

前号をコピーしてリネームします。

```bash
cp -r parent/202506-01 parent/202507-01
```

命名規則：`YYYYMM-NN`（例：`202507-01`）

### 2. 月次カバー画像を配置する

```bash
cp ~/Desktop/parent-202507-cover.png parent/202507-01/assets/img_cover.png
```

### 3. 号の中身を更新する

`parent/202507-01/index.html` を開き、月・本文・PDFリンクなどを更新します。

| 箇所 | 変更内容 |
|------|----------|
| `<title>` / OGタグ | 月・号数 |
| パス | `/parent/202507-01/...`（フォルダ名に合わせて自動で揃える） |
| `#letter` `#reading` `#column` `#notice` | 今月のコンテンツ |
| PDFリンク | `/pdfs/parent-202507-letter.pdf` など |

### 4. PDFを配置する

```
pdfs/parent-202507-letter.pdf
pdfs/parent-202507-reading-w1.pdf
…
pdfs/parent-202507-column.pdf
```

### 5. netlify.toml にリダイレクトを追加

### 6. アーカイブにカードを追加

`parent/index.html` の `issue-list` 内、**一番上**に `issue-card` を追加します。

```html
<a class="issue-card" href="/parent/202507-01">
  <div class="issue-card__thumb">
    <img src="/parent/202507-01/assets/img_cover.png" alt="" …>
  </div>
  …
</a>
```

### 7. デプロイ

```bash
git add parent/202507-01 parent/index.html netlify.toml pdfs/
git commit -m "publish parent letter 202507"
git push
```

## 各号のコンテンツ構成

| セクション | ID | 内容 |
|-----------|-----|------|
| その月の保護者通信 | `#letter` | 今月の学びの様子（PDF） |
| 英語読解特訓 | `#reading` | 長文解説PDF（Week 1〜N） |
| 保護者コラム | `#column` | 家庭での声かけ・見守り方 |
| お知らせ | `#notice` | 塾からのご案内 |

## 画像について

各号のトップ画像は **`parent/YYYYMM-01/assets/img_cover.png`**（1024×1024 推奨）。

アーカイブ上部のヒーローは **`parent/assets/img_archive_hero.png`**（全号共通）。

## トラブルシューティング

- **ページが 404** → `netlify.toml` のリダイレクトを確認
- **画像が表示されない** → パスが `/parent/202507-01/assets/...` 形式か確認
- **PDFが開けない** → `pdfs/` にファイルがあるか、HTML のリンクとファイル名が一致しているか確認
