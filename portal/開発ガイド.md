# 塾生ポータル 開発ガイド

このファイルは portal/ 配下を変更するAI(Cursor等)と開発者のための設計書。
作業前に必ず全文を読むこと。

## 0. 大原則

仕様の正典は「実際に動いているポータルとAPIとDB」である。
機能追加は必ず運営者(てった)の指示があったものだけを行う。
方針: 最低限から始めて、運用しながら一つずつ足す。

## 1. システム概要

塾生ポータルは、オンライン塾THINKING(合同会社ARC)の**塾生限定**の会員サイト。
club/(家勉部)や gyakuten-room/(逆転設計室)と同様、LIFF経由でLINE認証し、GAS APIで会員判定する。

全体は3層構造:

1. データ層: Googleスプレッドシート(未作成)
2. ロジック層: Google Apps Script をWebアプリとしてデプロイしたAPI(未作成)
3. 表示層: portal/index.html (このリポジトリ。Netlifyで thinking-online.com/portal/ に自動デプロイ)

参考となる既存ポータル:

| ディレクトリ | 対象 | URL |
|---|---|---|
| club/ | 家勉部員 | thinking-online.com/club/ |
| gyakuten-room/ | 逆転設計室 | thinking-online.com/gyakuten-room/ |
| portal/ | THINKING塾生 | thinking-online.com/portal/ |

## 2. 絶対に変更してはいけない箇所

portal/index.html の script 内にある以下は、デザイン変更の依頼であっても変更禁止:

- 定数 API_URL と LIFF_ID の値(設定後)
- liff.init / liff.login / liff.getIDToken / liff.getDecodedIDToken の呼び出しフロー
- api() 関数内の fetch のURL組み立て・メソッド
- APIに送るパラメータ名 (action, token)
- APIレスポンスのJSONキー名への依存箇所(キー名を別名に変えない)

POSTを使う場合はContent-Typeを指定しない(text/plainのまま)。指定するとCORSのpreflightが発生してGASで失敗する。

HTML構造とCSSは自由に変更してよい。ただし既存のid属性はJSが参照しているため、idを変更・削除する場合は参照しているJSも整合させること。

## 3. 認証とゲートの仕組み

1. ページを開くとLIFFが初期化され、LINEのIDトークンを取得
2. 全API呼び出しに token としてIDトークンを付与
3. GAS側がLINEの検証APIでトークンを検証し、line_user_idを特定
4. 会員マスタと照合し、ステータスで出し分ける
   - 有効: 全機能
   - 休会/退会/未登録: ゲート画面(JSONは {status:"denied", reason:"..."})
5. 未登録者のゲート画面には本人のLINE ID(sub)とコピー導線を表示する

## 4. API仕様(予定)

ベースURL(API_URL): GASのWebアプリ /exec

GET (クエリ: action, token):

- action=ping → {ok, version, time}
- action=me → {status:"ok", member:{display_name, grade, member_status, plan, ...}, contents:[{title, url, category}]}
- その他のアクションは運用開始時に追加

エラー系: {status:"unauthorized"} / {status:"denied", reason} / {status:"error", reason}

## 5. デザインシステム

ブランド: THINKING。トーン: ミニマルで知的。受験産業的な精神論・絵文字過多は避ける。

カラートークン(portal/assets/styles.css):

- --navy: #172642 (基調)
- --gold: #C8A55A (アクセント。多用しない)
- --paper: #F6F4EF (背景)
- --ink: #1C2433 / --muted: #6B7280 / --line: #E6E2D8

フォント: 本文 Noto Sans JP、数字 Space Grotesk(class="num")
レイアウト: モバイル前提、max-width 480px、カードUI

## 6. セットアップ手順(未完了項目)

1. LINE Developers で LIFF アプリを作成 → LIFF_ID を portal/index.html に設定
2. GAS で塾生ポータル用APIを作成・デプロイ → API_URL を portal/index.html に設定
3. 会員マスタ(スプレッドシート)を用意し、GASと連携
4. 公式LINEのリッチメニュー等から LIFF URL を配布

## 7. 更新フロー

見た目の変更: portal/index.html / portal/assets/styles.css を編集 → push → Netlifyが自動デプロイ(1〜2分)
機能の変更(API側): Apps Scriptエディタで編集 → デプロイを管理 → 新バージョンでデプロイ(URLは不変)

## 8. 動作確認

- 本番: スマホのLINEで LIFF URL を開く
- セットアップ前: LIFF_ID / API_URL が空のときは「準備中」ゲートが表示される
- ブラウザ単体テスト: GAS側にデバッグモードを設けた場合のみ ?debug_user= 形式でテスト可(本番では必ず無効化)
