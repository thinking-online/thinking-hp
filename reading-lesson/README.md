# 英文読解 完全解説（Reading Lesson）— 制作・検証ルール

`https://thinking-online.com/reading-lesson/<slug>` で公開する、大学入試 英文読解の解説教材。
**「問題文・設問を改変しない」「なぜ正解／なぜ不正解が原典・参考解説と齟齬しない」** を毎回保証するための運用ルールをここにまとめる。

---

## 1. フォルダ構成

```
reading-lesson/
  README.md                    ← このファイル（運用ルール）
  index.html                   ← 教材一覧（noindex）
  <slug>/
    index.html                 ← 教材本体（自己完結HTML・noindex）
    source/                    ← 照合の「正」= 原典と参考解説（Web非公開）
      passage.txt              ← 元の英文（逐語）
      questions.txt            ← 設問文・選択肢（逐語・並び順そのまま）
      answer-key.txt           ← 正解キー（問1=c … の一覧）
      reference.md / *.pdf     ← 参考解説（送ってもらった資料）
      notes.md                 ← 出典・年度・学部・大問番号などのメモ
```

- `source/` は `netlify.toml` で **Web非公開（404）** に設定済み。原典をリポジトリに残しつつ公開はしない。
- 教材本体・一覧ページは **`<meta name="robots" content="noindex, nofollow">`** を必ず入れる（受講生限定）。`sitemap.xml` には追加しない。

## 2. スラッグ（URL）命名規則

`<大学>-<学部>-<年度>[-qN]`（すべて小文字・ハイフン区切り）

| 例 | 意味 |
|---|---|
| `chuo-law-2023` | 中央大学 法学部 2023年 |
| `chuo-econ-2023` | 中央大学 経済学部 2023年 |
| `waseda-pol-2024-q2` | 早稲田 政経 2024年 大問2（同一試験で読解が複数あるとき `-qN`） |

- 学部を必ず入れる（同一大学・同一年度で衝突しないため）。
- 大問が1つだけなら `-qN` は省略。複数の読解を出すときだけ付ける。

## 3. 新しい問題を受け取ってから公開までの手順

**送ってもらうもの（3点セット）**
1. 元の英文（問題文）
2. 設問・選択肢（並び順そのまま）
3. 参考解説（正解と、なぜそうなるかの資料）

**制作フロー**
1. `reading-lesson/<slug>/source/` に上記を保存（原典を「正」として固定）。
2. 教材HTMLを制作 or 受領（Phase 1〜5 構成。テンプレは既存教材を流用）。
3. `noindex` メタ・タイトル・ヘッダーの大学名/年度を確認。
4. **↓ 検証チェックリスト（4章）を全項目パス**させる。
5. `index.html`（一覧）にカードを1枚追加。
6. `netlify.toml` は `:slug` ルールで自動対応するので **通常は追記不要**（`chuo2023` のような短縮エイリアスを別途貼りたい時だけ追記）。
7. コミット & push。`main` にマージされると本番反映。

## 4. 検証チェックリスト（毎回・全項目）

### A. 改変していないか（原典との逐語一致）
- [ ] 本文の英文が `source/passage.txt` と **1文字単位で一致**（Phase1 本文・Phase5 全文和訳の英文の両方）。空所 `( ア )` や下線の位置も原典どおり。
- [ ] 設問文・選択肢が `source/questions.txt` と一致。**選択肢の並び順（a/b/c/d）も原典どおり**。
- [ ] 語句の勝手な言い換え・省略・追加がない。

### B. 答えが正しいか
- [ ] 各設問の正解キー（HTML内 `s1..sN` の値）が `source/answer-key.txt` と一致。
- [ ] 「✓ 正解」表示と `sN` の値が食い違っていない。

### C. なぜ正解／なぜ不正解が妥当か
- [ ] 各設問に「なぜ正解か」＋「全選択肢の照合（なぜ不正解か）」がある。
- [ ] その論理が **参考解説（`source/reference`）と矛盾しない**。矛盾する場合は原典・参考解説を優先し、教材側を直す。
- [ ] 「根拠｜本文の該当箇所」の引用が本文に実在し、正解の根拠になっている。

### D. 内部整合
- [ ] Phase1 本文 ⇄ Phase5 全文和訳の英文が一致。
- [ ] 各設問の引用文（`.qen` など）が本文と一致。
- [ ] 「下線部・空所と設問の対応」表と実際の設問がずれていない。
- [ ] タブ数と `ORDER` 配列、`goP` / `sN` / `rN` の対応が揃っている（リンク切れ・未定義関数なし）。

### E. 表示・公開設定
- [ ] `noindex, nofollow` が入っている。
- [ ] `/reading-lesson/<slug>`（末尾スラッシュなし）で開ける。
- [ ] 一覧ページ `/reading-lesson` からリンクされている。

> 精度を上げたいときは、A〜D を独立した担当（照合／答え合わせ／論理検証／整合）に分けて並列でチェックする。
> Claude Code では「照合担当・論理検証担当・整合担当を分けて検証して」と頼めば複数エージェントで走らせられる。
